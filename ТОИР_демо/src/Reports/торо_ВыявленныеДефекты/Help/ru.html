Отчет «Выявленные дефекты»

Отчет предназначен для получения информации о выявленных дефектах объектов ремонта за период. Для доступа к отчету необходимо установить настройку системы в разделе Настройка параметров подсистемы ТОиР - Интерфейс и ввод документов - Учет показателей эксплуатации: Учет выявленных дефектов оборудования.
Открывается по команде Выявленные дефекты:

  • в списке Отчеты подсистемы Учет показателей эксплуатации;
  • в форме обработки Рабочее место диспетчера подсистемы Учет показателей эксплуатации.

Параметры отчета

Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. Для фильтра по дате обнаружения выявленных дефектов в отчете можно отредактировать параметр отчета: Период отчета.

Варианты отчета

С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Существует два предопределенных варианта отчета:

  1. Выявленные дефекты: таблица - открывается по умолчанию. Данные о выявленных дефектах объектов ремонта сгруппированы по объектам ремонта и видам дефектов.
    В этом варианте отчета по умолчанию в строках выводится информация о зарегистрированных в системе дефектах объектов ремонта. Регистрация дефектов объектов ремонта выполняется документом "Выявленные дефекты" или обработкой "Рабочее место диспетчера":
    • Объект ремонта - наименование объекта ремонта;
    • Вид дефекта - наименование вида дефекта объекта ремонта;
    • Документ - ссылка на документ "Выявленные дефекты";
    • Дата - дата и время обнаружения выявленного дефекта объекта ремонта;
    • Дефект выявлен в процессе - поле "Дефект выявлен в процессе" из документа "Выявленные дефекты";
    • Отказавший элемент - поле "Отказавший элемент" из документа "Выявленные дефекты";
    • Причина дефекта - поле "Причина дефекта" из документа "Выявленные дефекты";
    • Описание дефекта - поле "Описание дефекта" из документа "Выявленные дефекты";
    • Критичность дефекта - поле "Критичность дефекта" из документа "Выявленные дефекты";
    • Состояние дефекта - значение "Да" (значение поля отображается, если аварийный ремонт на основании документа "Выявленные дефекты" отменен через обработку "Закрытие заявок и ремонтов");
    • Причина закрытия - причина закрытия аварийного ремонта (значение поля отображается, если аварийный ремонт на основании документа "Выявленные дефекты" отменен через обработку "Закрытие заявок и ремонтов");
    • Оформленная заявка - одно из двух значений:
      1. Список ссылок на документы "Смета ремонта (Заявка на ремонт)", если на основании этого документа "Выявленные дефекты" введен один или несколько документов "Смета ремонта (Заявка на ремонт)".
      2. Значение "Нет оформленных заявок".
    Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.
  2. Выявленные дефекты: диаграмма - отчет предназначен для сравнительного анализа выявленных дефектов разных видов дефекта объектов ремонта в графическом виде.
