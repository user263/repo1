Отчет «Простой оборудования»

Отчет предназначен для получения информации о простоях объектов ремонта за период. Доступен при включенном флаге в разделе Настройка параметров подсистемы ТОиР - Интерфейс и ввод документов - Учет показателей эксплуатации: Учет состояния оборудования.
Открывается по команде Простой оборудования: в списке Отчеты подсистемы Учет показателей эксплуатации.

Параметры отчета

Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. Для фильтра по периоду действия простоев объектов ремонта в отчете можно отредактировать параметр отчета: Период отчета.

Варианты отчета

С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Существует два предопределенных варианта отчета:

  1. Простой оборудования: таблица - открывается по умолчанию. Данные о длительности простоев объектов ремонта сгруппированы по объектам ремонта и причинам простоев. Отчет формируется за период.
    В этом варианте отчета по умолчанию в строках выводится информация о зарегистрированных в системе простоях объектов ремонта. Регистрация простоев объектов ремонта выполняется документами "Состояния объектов ремонта" и "Ввод начальных данных":
    • Объект ремонта - наименование объекта ремонта;
    • Причина простоя - причина простоя объекта ремонта;
    • Дата останова - дата регистрации простоя оборудования. Заполняется одним из значений:
      1. Поле "Дата ввода в экспл." из карточки объектов ремонта, если при вводе начальных данных по объекту указывается состояние с типом "Простой".
      2. Поле "Дата изменения" из документов "Состояния объектов ремонта [Состояние на дату].
      3. Поле "Начало периода" из документов "Состояния объектов ремонта [Состояние на период]", если в документе "Состояния объектов ремонта [Состояние на период]" значение поля "Вид эксплуатации на период" в табличной части - это вид эксплуатации с типом "Простой".
      4. Поле "Окончание периода" из документов "Состояния объектов ремонта [Состояние на период]", если в документе "Состояния объектов ремонта [Состояние на период]" значение поля "Вид эксплуатации" в табличной части - это вид эксплуатации с типом "Простой".
    • Дата запуска - дата регистрации следующего (после данного простоя) изменения вида эксплуатации объекта ремонта. Заполняется одним из значений:
      1. Поле "Начало периода" из документов "Состояния объектов ремонта [Состояние на период]", если у объекта ремонта предыдущее состояние было с типом эксплуатации "Простой".
      2. Поле "Окончание периода" из документов "Состояния объектов ремонта [Состояние на период]", если в документе "Состояния объектов ремонта [Состояние на период]" значение поля "Вид эксплуатации на период" в табличной части - это вид эксплуатации с типом "Простой".
      3. Поле "Дата изменения состояния" из документов "Состояния объектов ремонта [Состояние на дату]", если у объекта ремонта предыдущее состояние было с типом эксплуатации "Простой".
      4. Пустое значение, если для объекта ремонта последним до текущего момента введен документ установки вида эксплуатации с типом эксплуатации "Простой".
    • Статус оборудования - состояние объекта ремонта на момент формирования отчета. У объекта ремонта может быть 3 состояния:
      1. Оборудование остановлено - для объекта ремонта последним до текущего момента введен документ установки вида эксплуатации с типом эксплуатации "Простой";
      2. Оборудование на испытаниях - для объекта ремонта последним до текущего момента введен документ установки вида эксплуатации с типом эксплуатации "Испытания";
      3. Оборудование работает - для объекта ремонта последним до текущего момента введен документ установки вида эксплуатации с типом эксплуатации "Эксплуатация".
    • Время простоя - время по графику работы объекта ремонта, которое объект ремонта стоял, выраженное в формате "ЧЧ:ММ:СС". Если оборудование на дату формирования отчета находится в простое, то время простоя рассчитывается от даты ввода оборудования в простой до даты окончания периода отчета (или до текущей даты, если период отчета не задан). 
    Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.
  2. Простой оборудования: диаграмма - отчет предназначен для получения информации о количестве объектов ремонта в каждой из трех групп состояний "Оборудование работает", "Оборудование остановлено", "Оборудование на испытаниях", представленной в графическом виде.
