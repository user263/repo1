Справочник «Списки объектов регламентных мероприятий»

В форме выводятся следующие поля:

  • Родитель – наименование группы элементов, в которую входит элемент справочника.
  • Наименование — название списка объектов регламентного мероприятия в справочнике.
  • Код — код списка объектов регламентного мероприятия.
  • Индивидуальное обслуживание ОР – если флажок установлен, то для каждого ОР из списка используется собственная технологическая карта, в противном случае - технологическая карта указанная для данного списка объектов регламентных мероприятий.
  • Количество ОР — автоматический счётчик количества введёных объектов ремонта.
  • Ремонтное подразделение — наименование подразделения организации, которое занимается обслуживанием объекта ремонта (указанное значение подставляется по умолчанию при оформлении документов).
  • Комментарий — дополнительная текстовая информация об списке регламентных мероприятий.

Закладка «Регламентные мероприятия и выходные документы»

  • Вид мероприятия – наименование вида ремонта.
  • Технологическая карта – наименование тех. карты мероприятия, в которой описывается состав и последовательность работ, их продолжительность и др. данные.
  • Способ планирования – наименование способа планирования, который будет использоваться при расчете графика ППР для определенного вида ремонта.
  • СМИ – Сокращать межремонтный интервал. Признак перемещения мероприятия на более раннюю дату, если дата ремонта попадает на выходные, в сторону сокращения межремонтного периода. Если перенос должен быть осуществлен на архивную дату – перемещение ремонта не выполняется. Если отметка не установлена, то при попадании следующих плановых ремонтов на выходные, при условии, что для этого вида ремонта ремонты на выходных не выполняются, начало ремонта переносится на первый рабочий день после выходных.


Подзакладка «Регламентные мероприятия и выходные документы»/«Предшествующие мероприятия»

  • Предшествующий вид мероприятия – наименование вида мероприятия, который при расчете графика регламентных мероприятий будет являться точкой отсчета для планируемого мероприятия.

Подзакладка  «Регламентные мероприятия и выходные документы»/«Дополнительно»

  • График ремонтных работ – наименование графика работы ремонтного персонала, в соответствии с которым производится планирование мероприятий.
  • Способ выполнения – наименование способа выполнения мероприятий: Хозспособ или Подрядный.
  • Способ учета даты отсчета – название способа, в соответствии с которым учитывается дата отсчета графика регламентных мероприятий.
  • Не использовать в планировании – признак того, что мероприятие не будет планироваться при формировании графика регламентных мероприятий и имеет справочное значение.
  • Не учитывать при планировании график ремонтных работ – признак того, что при формировании графика регламентных мероприятий ремонты будут запланированы без учета графика ремонтных работ оборудования, заданном в ремонтной группе объекта. Т.е. если мероприятия, например, согласно графику регламентных мероприятий в ремонтной группе не выполняются в выходные, то мероприятия на выходные все-равно будут запланированы – длительность мероприятий будет календарной.
  • Не учитывать продолжительность – признак, определяющий дату начала планирования следующего мероприятия. Если признак установлен, то в качестве даты следующего мероприятия определенного вида будет использована дата начала предыдущего. Если признак не установлен, то в качестве даты следующего мероприятия определенного вида будет использована дата дня, следующего за окончанием предыдущего.
  • Использовать коэффициенты ремонтных особенностей – признак того, что при формировании графика регламентных мероприятий будут учитываться заданные коэффициенты ремонтных особенностей.

С помощью кнопки [Ввести продолжительность] можно установить продолжительность мероприятия для списка оборудования по нормативам объектов.

Подзакладка «Регламентные мероприятия и выходные документы»/«Выходные документы»

  • Выходной документ - перечисление типов выходных документов, оформляемых во время или по итогам мероприятия.

 

Закладка «Список объектов»

  • №п/п – номер объекта ремонта по порядку обхода маршрута.
  • Код – код объекта ремонта, заполняется автоматически при добавлении объекта ремонта в маршрут.
  • Объект ремонта – объект ремонта, входящий в состав маршрута.
  • Местонахождение – местонахождение объекта ремонта, заполняется автоматически при добавлении объекта ремонта в маршрут.
