Справочник «Типовые объекты ремонта»

В форме выводятся следующие поля:

  • Родитель – наименование группы элементов, в которую входит элемент справочника.
  • Наименование – краткое название элемента справочника.
  • Код – код элемента справочника.
  • Направление - направление типового объекта ремонтных работ.
  • Изготовитель - наименование изготовителя.
  • Комментарий - свободное текстовое поле. 

 

Закладка «Нормативные ремонты»

  • Вид ремонта – наименование вида ремонта.
  • Технологическая карта ремонта – наименование технологической карты ремонта, в которой описывается состав и последовательность работ, их продолжительность и др. данные.
  • Способ планирования – наименование способа планирования, который будет использоваться при расчете графика ППР для указанного вида ремонта.

 

Подзакладка «Нормативные ремонты»/«Предшествующие ремонты»

  • Предшествующий вид ремонта – наименование вида ремонта, который при расчете графика ППР будет являться точкой отсчета для планируемого ТО или ремонта.

 

Подзакладка «Нормативные ремонты»/«Замещаемые ремонты»

  • Вид замещаемого ремонта – наименование вида ремонтных работ, который будет замещаться текущим видом ремонта.
  • Способ замещения – название одного из предопределенных в системе способов замещения, указывающего на то, каким образом при расчете графика ППР будет производится замещение.
  • Дней от начала замещающего ремонта (влево)– значение, указывающее на количество дней до начала выполнения замещаемого вида ремонтных работ (при попадании замещающего вида ремонта в этот период выполняется замещение).
  • Дней от окончания замещающего ремонта (вправо)— значение, указывающее на количество дней после окончания выполнения замещаемого вида ремонтных работ (при попадании замещающего вида ремонта в этот период выполняется замещение).

 

Подзакладка «Нормативные ремонты»/«Дополнительно»

  • График выполнения ремонтных работ – наименование графика работы ремонтного персонала, в соответствии с которым производится планирование ТО и ремонтов.
  • Способ выполнения – наименование способа выполнения ремонтных работ: Хозспособ или Подрядный.
  • Способ учета даты отсчета – название способа, в соответствии с которым учитывается дата отсчета графика ППР: По дате ввода в эксплуатацию или последнему ремонту; Использовать дату планирования; По последнему ремонту.
  • Не использовать в планировании – признак того, что нормативный ремонт не будет планироваться при формировании графика ППР и будет иметь справочное значение.
  • Не учитывать при планировании график работы оборудования – признак, определяющий способ расчета плановой наработки для определения даты следующего ремонта при расчете графика ППР. Если признак не установлен, то расчет плановой наработки будет выполняться с учетом графика работы оборудования, т.е., если оборудование не работает в выходные, считается, что у него в выходные наработки нет, а в остальные дни наработка будет равна указанной для объекта ремонта в закладке “Наработка. Если признак установлен, то расчет плановой наработки будет выполняться без учета графика работы оборудования, и наработка за любой день будет одинакова и равна указанной для объекта ремонта в закладке “Наработка”.
  • Не учитывать при планировании график ремонтных работ – признак того, что при формировании графика ППР ремонты будут запланированы без учета графика ремонтов оборудования, заданном в ремонтной группе объекта. Т.е., если ремонты, например, согласно графику ремонтных работ в ремонтной группе не выполняются в выходные, то ремонты на выходные все равно будут запланированы – длительность ремонтов будет календарной.
  • Не учитывать продолжительность – признак, определяющий дату начала планирования следующего ремонта. Если признак установлен, то в качестве даты следующего ремонта определенного вида будет использована дата начала предыдущего. Если признак не установлен, то в качестве даты следующего ремонта определенного вида будет использована дата дня, следующего за окончанием предыдущего.
  • Учитывать простои оборудования – признак того, что при формировании графика ППР ремонты будут запланированы с учетом простоев оборудования.
  • Использовать коэффициенты ремонтных особенностей (отображается при установке соответствующего признака в настройках параметров подсистемы "Управление ремонтами", закладка Учёт активов) – признак того, что при формировании графика ППР будут учитываться заданные коэффициенты ремонтных особенностей.
  • Не использовать в планировании дату ввода в эксплуатацию - признак того, что при формировании графика ППР в качестве даты отсчёта не будет браться дата ввода в эксплуатацию.
  • Разрешать сокращать межремонтный период при планировании – признак перемещения планового ремонта на более раннюю дату, если дата ремонта попадает на выходные, в сторону сокращения межремонтного периода. Если перенос должен быть осуществлен на архивную дату – перемещение ремонта не выполняется. Если отметка не установлена, то при попадании следующих плановых ремонтов на выходные, при условии, что для этого вида ремонта ремонты на выходных не выполняются, начало ремонта переносится на первый рабочий день после выходных.

 

Закладка «Состав»

  • Код – код объекта ремонта.
  • Наименование – название объекта ремонта.

 

Закладка «Показатели»

  • Показатель — наименование контролируемого показателя.
  • Ед. изм. – единица измерения контролируемого показателя.
  • Из типового ОР — значение контролируемого показателя.
  • Доп. минимум – минимальное допустимое значение контролируемого показателя.
  • Доп. максимум – максимальное допустимое значение контролируемого показателя.
  • Крит. минимум – минимальное критическое значение контролируемого показателя. В случае, если значение контролируемого показателя ниже критического минимума, необходим внеплановый ремонт оборудования для предотвращения его аварийного выхода из строя.
  • Крит. максимум – максимальное критическое значение контролируемого показателя, при превышении которого необходим внеплановый ремонт оборудования для предотвращения его аварийного выхода из строя.
  • Вид сравнения – список вариантов отбора.
  • Вид значения – список видов значения контролируемых показателей.
  • Значение – необходимое для сравнения значение контролируемого показателя.
  • Параметры уведомления –

 

Закладка «Запчасти»

  • Номенклатура – наименование единицы номенклатуры, которой соответствует запасная часть.
  • Количество – количество единиц номенклатуры в указанных ед. измерения, необходимых для выполнения ТО и ремонтов для объекта ремонта.
  • Характеристика – характеристика номенклатуры.
  • Качество – значение, определяющее качество номенклатуры.

 

Подзакладка «Запчасти»/«Нормативные затраты при ТОиР»

  • Вид ремонта – наименование вида ремонта, при выполнении которого используется запасная часть.
  • Количество – количество единиц номенклатуры в указанных ед. измерения, используемых в указанном виде ремонта.

 

Закладка «Оценка РМ»

  • Дата – дата ввода оценки риск-менеджмента.
  • Вид ремонта – вид ремонта, для которого выполняется оценка риск-менеджмента.
  • Сезон – годовой сезон выполнения ремонтов.
  • Вероятность выхода из строя, % – оцениваемая вероятность выхода из строя в процентах при невыполнении данного вида ремонта
  • Ущерб – денежный эквивалент ущерба при невыполнении данного вида ремонта.

 

Закладка «Структура ремонтного цикла»

  • Начало цикла – вид ремонта, являющийся началом и концом ремонтного цикла.
  • Плановый график – график работы оборудования.

С помощью кнопки [Построить/Обновить] можно вывести графическое представление ремонтного цикла.

С помощью кнопки [Настройки] можно вывести форму выбора видов ремонтов для отображения.

С помощью кнопки [Перерассчитать] можно выполнить перерасчет визуализации ремонтного цикла.

 

Закладка «Наработка»

Группа полей "Показатели плановой наработки" содержит перечень показателей, для которых в системе регистрируется наработка объекта ремонта.

  • Дата введения – дата введения показателя плановой наработки.
  • Показатель наработки – наименование показателя, по которому фиксируется наработка текущего объекта ремонта.
  • Ед. изм. – единица, в которой измеряется наработка объекта ремонта для текущего показателя.
  • Продолжительность периода плановой наработки – промежуток времени, на который вводится плановое значение наработки. Значение выбирается из выпадающего списка и может содержать одно из предопределенных значений: День, Неделя, Декада, Месяц, Квартал, Полугодие, Год.
  • Плановая наработка – значение, которое будет использоваться для расчета наработки объекта ремонта в случае отсутствия фактических значений в журнале наработки.
  • К-т использования – коэффициент учета плановой наработки. Коэффициент учитывает долю аварийных ремонтов при планировании по наработке.

Группа полей "Максимальные значения показателей наработки"

  • Показатель наработки — наименование показателя наработки.
  • Макс. значение — значение максимального показателя наработки.
